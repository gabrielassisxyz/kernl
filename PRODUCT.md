# Product

## Register

product

## Users
Solo developers who build their own tools, use agents to accelerate execution, and feel the pain of context fragmentation across multiple tools (notes, tasks, AI chat). They are highly technical, operate primarily on desktop Linux, and value keyboard-driven efficiency.

## Product Purpose
Kernl is the solo developer's cognitive substrate. It unifies knowledge (notes, bookmarks), project management (tasks, epics), and multi-agent execution into a single local-first knowledge graph. It exists so that judgment becomes the only thing the human does, while decomposition and execution run in parallel without requiring continuous supervision.

## Brand Personality
Focused, utilitarian, structured. The interface should feel like a native, responsive tool that stays out of the way so complex data and workflows can shine. References: Linear (for high data density without clutter and excellent dark mode), Obsidian (for the local-first "vault" aesthetic), and Ghostty/Raycast (for raw, keyboard-driven utility and speed).

## Anti-references
- Jira: Bureaucratic, cluttered, slow, and overly enterprise-focused.
- Notion: Too much whitespace, overly document-centric, too "soft" or generic.
- Generic SaaS Dashboards: Avoid decorative gradients, glassmorphism, oversized colorful cards, and exaggerated drop shadows. The design should serve the data, not decorate it.

## Design Principles
1. **Information density with clarity:** Present multiple interconnected data points (tasks, notes, agents) simultaneously without overwhelming the user.
2. **Execution over presentation:** The UI exists to get work done and observe autonomous execution, not just to look pretty.
3. **Respect the developer's environment:** Dark theme first, built for large desktop monitors, prioritizing structural hierarchy and speed over flashy animations.
4. **Substrate cohesion:** Visually reinforce that everything (a note, a task, an agent session) belongs to the same interconnected graph.

## Accessibility & Inclusion
- High contrast, particularly for text on dark backgrounds.
- Clear structural layout to support information density.
- Keyboard-accessible navigation.
- Reduced motion support (critical for a utilitarian tool).
