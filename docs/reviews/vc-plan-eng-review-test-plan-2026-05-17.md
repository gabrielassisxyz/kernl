## Affected packages / surfaces
- `internal/graph` — top-level Open/Close, Tx types, DoRead/DoWrite
- `internal/graph/nodes` — NodeSpec, chokepoint (createNode/updateNode/deleteNode), per-type filters
- `internal/graph/edges` — Create/Delete/Outgoing/Incoming

## Key interactions to verify

### Chokepoint invariants
- Every Create/Update/Delete writes exactly one revision row in the same tx.
- Every Create/Update/Delete updates the FTS index in the same tx (state matches after).
- Tags reconciliation (insert added, delete removed) happens in the same tx.
- Author is required (empty string → reject at chokepoint, not DB).

### Type-distinct transactions
- `UpdateBead(tx ReadTx, ...)` must fail to compile (compile-fail testdata).
- `ReadTx` and `WriteTx` cannot be constructed outside `internal/graph` (unexported method on interface).

### Identity
- UUIDv7s generated within a process are monotonically increasing.
- UUIDv7s sorted lexicographically equal UUIDs sorted by creation time (proves index-locality claim).

### Persistence
- Close → re-Open → all data persists (tempfile mode).
- Migration up → down → up round-trip preserves schema (CI gate).
- InMemory mode applies migrations identically.

## Edge cases

### FTS5
- Special characters in indexed body (`"`, `*`, `^`, `~`, `:`) — insert + search round-trip without panic.
- Malformed FTS5 query → wrapped error (`graph: search: query syntax: ...`), no raw FTS5 internals leaked.
- Empty query → documented behavior (probably error).
- Single-character query → returns expected (or none) without panic.
- Maximum-length query (e.g., 10k chars) — does not crash.
- Portuguese accents — "ação"/"coração" → searching "acao" returns hits with `remove_diacritics 2`.
- Mixed-language body (EN + PT) → both languages searchable.

### Edges
- Self-edge (src == dst) — allowed at schema level; documented; tested both directions of Outgoing/Incoming.
- Edge to non-existent dst → FK violation (proves `foreign_keys=ON`).
- Cascade: delete src node → all its outgoing edges gone.
- Cascade: delete dst node → all its incoming edges gone.

### Revisions
- **CRITICAL (regression-style for the schema correction):** Delete node → revisions table still contains all prior revisions + the tombstone. Walk back via `prev_revision_id` reaches the create row.
- `prev_revision_id` chain unbroken across N updates.
- Tombstone author preserved post-delete.
- Concurrent updaters: revision rows are correctly ordered by `ts` even when wall-clock collides (UUIDv7 ordering is the tiebreaker).

### Concurrency
- Two goroutines call `DoWrite` simultaneously → both succeed serially via write-pool `SetMaxOpenConns(1)`; no `SQLITE_BUSY` observable to callers.
- Reader holding a `ReadTx` against WAL sees pre-delete snapshot of a node concurrently deleted by another goroutine.

### Boundary inputs
- Node with empty `title` (default '').
- Node with `attrs = '{}'` (default).
- Tag with empty name → reject at API (or document allow).
- Bulk: 1000 nodes inserted sequentially → query latency stays sub-ms.

## Critical paths (must-work end-to-end)

1. **Bead lifecycle:** Open → CreateBead → GetBead → UpdateBead → revision count is 2 → DeleteBead → tombstone exists, prior rows preserved.
2. **Note + FTS:** CreateNote with body → Search returns it → UpdateNote with new body → Search for old word returns empty, new word returns the note.
3. **Edge cascade:** Create beadA + beadB + edge(A→B, depends_on) → Outgoing(A) returns edge → DeleteNode(A) → edge removed; revisions of A preserved (no FK on revisions.node_id).
4. **Tags many-to-many:** AddTag(noteX, "y") → tags.Nodes("y") returns X → RemoveTag → empty.
5. **Persistence:** Open(tempfile) → CRUD → Close → Open → same data → FTS still queryable.
6. **Property test (rapid):** 1000 random ops on a mix of nodes/edges/tags. Invariants checked after every op:
   - Exactly one new revision row per mutation.
   - FTS reflects current state (no stale entries).
   - Cascade removes edges/tags/fts column from `nodes` for deleted nodes.
   - Revisions are NOT removed by cascade.
   - UUIDv7 monotonic.

## Test execution
- All tests run under `go test ./internal/graph/... -race`.
- Property tests use `pgregory.net/rapid` (new dep — small, shrinking, no transitive baggage).
- CI gate: migration up → down → up round-trip on every push.
